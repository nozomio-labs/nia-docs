Getting Started
---------------

<Steps>
  <Step title="Sign Up">
    1. Visit [Nia AI](https://app.trynia.ai)

    2. Click "Sign In" or "Sign Up"

    3. Use your GitHub account or email for authentication

    <Tip>
      Using GitHub authentication gives you immediate access to your repositories.
    </Tip>
  </Step>

  <Step title="Create Your First Project">
    After logging in:

    1. Click "New Project" button

    2. Enter a project name (e.g., "My Backend API")

    3. Select the repository you want to analyze

    <Note>
      You can connect multiple repositories to different projects to keep your codebases organized.
    </Note>
  </Step>

  <Step title="Index Your Repository">
    ```bash
    # You can specify:
    - Main branch (default)
    - Specific branch
    - Commit hash
    ```

    <Warning>
      Initial indexing might take a few minutes depending on repository size.
    </Warning>
  </Step>
</Steps>

## Asking Questions

<CardGroup cols={2}>
  <Card title="Effective Questions">
    ✅ **Do Ask About**:

    * Code architecture

    * Implementation details

    * Function purposes

    * Dependencies

    * Best practices
  </Card>

  <Card title="Less Effective Questions">
    ❌ **Avoid Asking**:

    * Runtime errors

    * System-specific issues

    * Questions without context

    * Non-code related queries
  </Card>
</CardGroup>

### Question Examples

<AccordionGroup>
  <Accordion title="Code Understanding">
    ```markdown
    # Great Questions:
    - "How is user authentication implemented in this codebase?"
    - "What's the data flow for processing user requests?"
    - "Explain the error handling strategy in the API endpoints."
    ```
  </Accordion>

  <Accordion title="Implementation Details">
    ```markdown
    # Specific Queries:
    - "Show me how API rate limiting is implemented."
    - "What validation is used for user input?"
    - "How are database connections managed?"
    ```
  </Accordion>

  <Accordion title="Architecture Questions">
    ```markdown
    # System Design:
    - "Explain the project's folder structure and organization."
    - "How are environment variables managed?"
    - "What's the deployment pipeline look like?"
    ```
  </Accordion>
</AccordionGroup>

## Best Practices

### Project Organization

<Steps>
  <Step title="Separate Projects">
    Create different projects for:

    * Frontend codebase

    * Backend services

    * Infrastructure code
  </Step>

  <Step title="Regular Indexing">
    Re-index when you:

    * Make major changes

    * Switch branches

    * Pull updates
  </Step>

  <Step title="Chat Organization">
    * Use descriptive chat titles

    * Keep related questions in same chat

    * Create new chats for different topics
  </Step>
</Steps>

## Common Questions

<AccordionGroup>
  <Accordion title="How often should I re-index?">
    Re-index your repository when:

    * Making significant code changes

    * Switching to a different branch

    * After merging important PRs
  </Accordion>

  <Accordion title="What's the best way to start?">
    1. Start with broad questions about architecture

    2. Drill down into specific components

    3. Ask for examples and explanations
  </Accordion>

  <Accordion title="Can I ask about multiple files?">
    Yes! Nia AI understands relationships between files. Reference them in your question:
    "How does auth.ts interact with the user service?"
  </Accordion>
</AccordionGroup>
