<Steps>
  <Step title="Complete Onboarding">
    Visit [app.trynia.ai](https://app.trynia.ai) and complete the onboarding process. Your API key will be automatically created.

    <Info>
      You'll get 3 free index jobs to start - no credit card required!
    </Info>
  </Step>

  <Step title="Run Automated Setup">
    Open your terminal and run the setup script with your API key and coding agent:

    ```bash
    curl -fsSL http://app.trynia.ai/api/setup-script | bash -s -- YOUR_API_KEY your-coding-agent
    ```

    Replace:
    - `YOUR_API_KEY` with your API key from step 1
    - `your-coding-agent` with one of: `cursor`, `claude-code`, `vscode`, `windsurf`, `continue`, `cline`, `codex`, `trae`, `antigravity`

    **Examples:**
    ```bash
    # For Cursor
    curl -fsSL http://app.trynia.ai/api/setup-script | bash -s -- nia_abc123 cursor

    # For Claude Code
    curl -fsSL http://app.trynia.ai/api/setup-script | bash -s -- nia_abc123 claude-code

    # For VS Code
    curl -fsSL http://app.trynia.ai/api/setup-script | bash -s -- nia_abc123 vscode
    ```

    <Tip>
      This script will automatically install dependencies and configure Nia MCP for your coding agent.
    </Tip>
  </Step>

  <Step title="Restart Your Coding Agent">
    Restart your coding assistant to load the new MCP server
  </Step>
</Steps>

## That's It!

You're ready to start using Nia MCP! Check out the [Tools & Features](/tools-features) page to explore all available capabilities.

<Note>
  **Prefer manual setup?** See the [Manual Setup](#manual-setup) section below for detailed configuration instructions.
</Note>

## Manual Setup

If you prefer to manually configure Nia MCP or if the automated setup didn't work for your setup, follow these detailed instructions:

<Steps>
  <Step title="Prerequisites">
    - Python 3.8+
    - `pipx` (recommended) or `uvx` or `pip`
    - Your Nia API key from [app.trynia.ai](http://app.trynia.ai/)
  </Step>

  <Step title="Install the Package">
    <CodeGroup>
      ```bash Option A: Using pipx (Recommended)
      # Install globally for use across all projects
      pipx install nia-mcp-server
      ```

      ```bash Option B: Using uvx
      # uvx will automatically handle the installation when you run it
      # No need to pre-install if you have uvx
      ```

      ```bash Option C: Using pip (If needed)
      pip install nia-mcp-server
      ```
    </CodeGroup>
  </Step>

  <Step title="Configure Your MCP Client">
    Choose your coding agent below for specific configuration steps
  </Step>

  <Step title="Replace Your API Key">
    Get your API key from [app.trynia.ai](http://app.trynia.ai/) and replace `YOUR_API_KEY` in the configuration
  </Step>

  <Step title="Restart Your Client">
    Restart your coding assistant to load the new MCP server
  </Step>
</Steps>

  <Tab title="Claude Code">
    <Steps>
      <Step title="Add NIA Configuration">
        <CodeGroup>
          ```bash Option A: Using pipx (Recommended)
          # For current project only (default)
          claude mcp add nia -e NIA_API_KEY=YOUR_API_KEY -e NIA_API_URL=https://apigcp.trynia.ai/ -- pipx run --no-cache nia-mcp-server

          # For global installation (all projects)
          claude mcp add nia -e NIA_API_KEY=YOUR_API_KEY -e NIA_API_URL=https://apigcp.trynia.ai/ --scope user -- pipx run --no-cache nia-mcp-server
          ```

          ```bash Option B: Using uvx
          # For current project only (default)
          claude mcp add nia -e NIA_API_KEY=YOUR_API_KEY -e NIA_API_URL=https://apigcp.trynia.ai/ -- uvx nia-mcp-server

          # For global installation (all projects)
          claude mcp add nia -e NIA_API_KEY=YOUR_API_KEY -e NIA_API_URL=https://apigcp.trynia.ai/ --scope user -- uvx nia-mcp-server
          ```
        </CodeGroup>

        <Note>
          **Scope Options:**
          - **Default (no --scope flag)**: Installs for current project only
          - **--scope user**: Installs globally for all projects on your machine

          Use `--scope user` if you want Nia available across all your projects. This is ideal for personal development where you frequently use Nia across different codebases.
        </Note>

      <Step title="Configure Timeouts (Important!)">
        <Warning>
          **Codex requires extended timeout settings for NIA to work properly.** Without these settings, tool outputs may get truncated or timeout prematurely.
        </Warning>

        Edit your `.codex/config.toml` file and add the following configuration:

        ```toml
        tool_output_token_limit = 25000  # Extended cap to prevent output truncation

        [mcp_servers.nia]
        command = "pipx"
        args = ["run", "--no-cache", "nia-mcp-server"]
        startup_timeout_sec = 30
        tool_timeout_sec = 360  # 6 minutes - essential for NIA operations
        ```

        <Note>
          **Why these settings matter:**
          - `tool_output_token_limit = 25000`: Prevents NIA's comprehensive responses from being truncated
          - `tool_timeout_sec = 360`: Gives NIA enough time to complete indexing and search operations (6 minutes is plenty)
          - `startup_timeout_sec = 30`: Ensures NIA server has time to initialize properly
        </Note>
      </Step>
