## üì¶ Installation

<Steps>
  <Step title="Prerequisites">
    - Python 3.8+ 
    - `pipx` (recommended) or `uvx` or `pip`
    - Your NIA API key from [app.trynia.ai](http://app.trynia.ai/)
  </Step>
  
  <Step title="Install the Package">
    <CodeGroup>
      ```bash Option A: Using pipx (Recommended)
      # Install globally for use across all projects
      pipx install nia-mcp-server
      ```
      
      ```bash Option B: Using uvx
      # uvx will automatically handle the installation when you run it
      # No need to pre-install if you have uvx
      ```
      
      ```bash Option C: Using pip (If needed)
      pip install nia-mcp-server
      ```
    </CodeGroup>
  </Step>
  
  <Step title="Configure Your MCP Client">
    Choose your coding assistant below for specific configuration steps
  </Step>
  
  <Step title="Replace Your API Key">
    Get your API key from [app.trynia.ai](http://app.trynia.ai/) and replace `YOUR_API_KEY` in the configuration
  </Step>
  
  <Step title="Restart Your Client">
    Restart your coding assistant to load the new MCP server
  </Step>
</Steps>

## üìã Complete Tool Reference

<AccordionGroup>
  <Accordion title="Repository Management Tools">
    ### `index_repository`
    Index a GitHub repository for intelligent code search.
    
    **Parameters**:
    - `repo_url` (str): GitHub repository URL (e.g., https://github.com/owner/repo)
    - `branch` (str, optional): Branch to index (defaults to main branch)
    
    **Usage**: When indexing starts, use `check_repository_status` to monitor progress.

    ### `list_repositories`
    List all indexed repositories with their status.
    
    **Parameters**: None
    
    **Returns**: List of repositories with status, branch, and indexing information.

    ### `check_repository_status`
    Check the indexing status of a repository.
    
    **Parameters**:
    - `repository` (str): Repository in owner/repo format
    
    **Returns**: Current status, progress, and indexing details.

    ### `delete_repository`
    Delete an indexed repository.
    
    **Parameters**:
    - `repository` (str): Repository in owner/repo format
    
    **Returns**: Confirmation of deletion.

    ### `rename_repository`
    Rename an indexed repository for better organization.
    
    **Parameters**:
    - `repository` (str): Repository in owner/repo format
    - `new_name` (str): New display name (1-100 characters)
    
    **Returns**: Confirmation of rename operation.

    ### `search_codebase`
    Search indexed repositories using natural language.
    
    **Parameters**:
    - `query` (str): Natural language search query (use comprehensive questions for best results)
    - `repositories` (List[str], optional): List of repositories to search (owner/repo format)
    - `include_sources` (bool, default=True): Whether to include source code in results
    
    **Returns**: Search results with relevant code snippets and explanations.
  </Accordion>

  <Accordion title="Documentation Management Tools">
    ### `index_documentation`
    Index documentation or website for intelligent search.
    
    **Parameters**:
    - `url` (str): URL of the documentation site to index
    - `url_patterns` (List[str], optional): URL patterns to include in crawling (e.g., ["/docs/*", "/guide/*"])
    - `max_age` (int, optional): Maximum age of cached content in seconds
    - `only_main_content` (bool, default=True): Extract only main content (removes navigation, ads, etc.)
    
    **Usage**: When indexing starts, use `check_documentation_status` to monitor progress.

    ### `list_documentation`
    List all indexed documentation sources.
    
    **Parameters**: None
    
    **Returns**: List of documentation sources with status and metadata.

    ### `check_documentation_status`
    Check the indexing status of a documentation source.
    
    **Parameters**:
    - `source_id` (str): Documentation source ID
    
    **Returns**: Current status, progress, and indexing details.

    ### `delete_documentation`
    Delete an indexed documentation source.
    
    **Parameters**:
    - `source_id` (str): Documentation source ID to delete
    
    **Returns**: Confirmation of deletion.

    ### `rename_documentation`
    Rename a documentation source for better organization.
    
    **Parameters**:
    - `source_id` (str): Documentation source ID
    - `new_name` (str): New display name (1-100 characters)
    
    **Returns**: Confirmation of rename operation.

    ### `search_documentation`
    Search indexed documentation using natural language.
    
    **Parameters**:
    - `query` (str): Natural language search query (use comprehensive questions for best results)
    - `sources` (List[str], optional): List of documentation source IDs to search
    - `include_sources` (bool, default=True): Whether to include source references in results
    
    **Returns**: Search results with relevant documentation excerpts.
  </Accordion>

  <Accordion title="Web Search & Research Tools">
    ### `nia_web_search`
    Search repositories, documentation, and other content using AI-powered search.
    
    **Use Cases**:
    - Finding specific repos/docs/content
    - Looking for examples or implementations
    - Searching for what's available on a topic
    - Simple, direct searches that need quick results
    - Finding similar content to a known URL
    
    **Parameters**:
    - `query` (str): Natural language search query
    - `num_results` (int, default=5): Number of results to return (max: 10)
    - `category` (str, optional): Filter by category ("github", "company", "research paper", "news", "tweet", "pdf")
    - `days_back` (int, optional): Only show results from the last N days
    - `find_similar_to` (str, optional): URL to find similar content to
    
    **Returns**: Search results with actionable next steps.

    ### `nia_deep_research_agent`
    Perform deep, multi-step research on a topic using advanced AI research capabilities.
    
    **Use Cases**:
    - Comparing multiple options ("compare X vs Y vs Z")
    - Analyzing pros and cons
    - Questions with "best", "top", "which is better"
    - Needing structured analysis or synthesis
    - Complex questions requiring multiple sources
    - Questions about trends, patterns, or developments
    - Requests for comprehensive overviews
    
    **Parameters**:
    - `query` (str): Research question (use comprehensive questions for best results)
    - `output_format` (str, optional): Structure hint (e.g., "comparison table", "pros and cons list")
    
    **Returns**: Comprehensive research results with citations.

    ### Tool Selection Guide

    **Use `nia_web_search` for**:
    - "Find RAG libraries" ‚Üí Simple search
    - "What's trending in Rust?" ‚Üí Quick discovery
    - "Show me repos like LangChain" ‚Üí Similarity search

    **Use `nia_deep_research_agent` for**:
    - "Compare RAG vs GraphRAG approaches" ‚Üí Comparative analysis
    - "What are the best vector databases for production?" ‚Üí Evaluation needed
    - "Analyze the pros and cons of different LLM frameworks" ‚Üí Structured analysis
  </Accordion>

  <Accordion title="Project Initialization Tools">
    ### `initialize_project`
    Initialize a NIA-enabled project with IDE-specific rules and configurations.
    
    **Parameters**:
    - `project_root` (str): Absolute path to the project root directory
    - `profiles` (List[str], optional): List of IDE profiles to set up (default: ["cursor"])
    
    **Supported Profiles**: cursor, vscode, claude, windsurf, cline, codex, zed, jetbrains, neovim, sublime
    
    **Returns**: Status of initialization with created files and next steps.
    
    **Examples**:
    - Basic: `initialize_project("/path/to/project")`
    - Multiple IDEs: `initialize_project("/path/to/project", profiles=["cursor", "vscode"])`
    - Specific IDE: `initialize_project("/path/to/project", profiles=["windsurf"])`
  </Accordion>
</AccordionGroup>

## ‚ö†Ô∏è Troubleshooting

<AccordionGroup>
  <Accordion title='"No API key provided"'>
    **Solution:** Make sure you've replaced `YOUR_API_KEY` with your actual API key from [app.trynia.ai](http://app.trynia.ai/)
    
    Double-check the environment variable is correctly set in your MCP configuration.
  </Accordion>
  
  <Accordion title='"Command not found: nia-mcp-server"'>
    **Solution:** Try installing with pipx first:
    ```bash
    pipx install nia-mcp-server
    ```
    
    Or ensure `pipx` is properly installed:
    ```bash
    pip install pipx
    ```
    
    Alternatively, you can use `uvx` or regular `pip` installation.
  </Accordion>
  
  <Accordion title='"Invalid API key"'>
    **Solution:** 
    - Double-check your API key at [app.trynia.ai](http://app.trynia.ai/)
    - Ensure there are no extra spaces or characters
    - Verify the API endpoint is set to `https://apigcp.trynia.ai/`
  </Accordion>
  
  <Accordion title='"Rate limit exceeded"'>
    **Solution:** You've used your 25 free requests. Check your usage and upgrade your plan at [app.trynia.ai](http://app.trynia.ai/)
  </Accordion>
  
  <Accordion title="MCP Server Not Responding">
    **Solution:**
    1. Check if the server is running properly
    2. Verify your MCP client configuration
    3. Restart your coding assistant
    4. Check the console for error messages
  </Accordion>
</AccordionGroup>
