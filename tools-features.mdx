## Capability Overview

<CardGroup cols={2}>
  <Card title="Index & Subscribe" icon="database">
    Bring knowledge into Nia from repositories, docs, papers, datasets, spreadsheets, Google Drive, and local folders. Use `index`, `manage_resource`, and `auto_subscribe_dependencies`.
  </Card>

  <Card title="Search, Read & Explore" icon="magnifying-glass">
    Search semantically, grep precisely, read files and sections, and inspect structures. Use `search`, `nia_read`, `nia_grep`, `nia_explore`, and `nia_package_search_hybrid`.
  </Card>

  <Card title="Research & Analysis" icon="flask">
    Ask Nia to compare options, investigate a problem, or review your code against documentation. Use `nia_research`, `nia_advisor`, [Tracer](/tracer), and [Oracle Research Agent](/oracle-research).
  </Card>

  <Card title="Search Without Indexing" icon="code-branch">
    Need quick answers from public code right now? Use [Tracer](/tracer) for GitHub repositories and `nia_package_search_hybrid` for package source code.
  </Card>

  <Card title="Local Sync" icon="folder-sync">
    Keep local folders, databases, and chat history fresh in Nia with continuous synchronization. Start with [Local Sync](/local-sync).
  </Card>

  <Card title="Vaults" icon="book-sparkles" href="/vaults">
    Agent-maintained personal wikis that compound knowledge from your indexed sources. Concepts, entities, wikilinks, dream cycles, and a knowledge graph — all built automatically.
  </Card>

  <Card title="Context Sharing" icon="comments">
    Save plans, discoveries, edited files, and conversation state so another agent can pick up where you left off. Use `context`.
  </Card>

  <Card title="Scoped MCP" icon="server">
    Create a smaller, task-specific MCP surface for a particular source or workflow. Start with [Scoped MCP Servers](/scoped-mcp).
  </Card>

  <Card title="Explore Pre-indexed Knowledge" icon="globe">
    Ask questions across existing indexed sources without setting up everything yourself. Use [Explore & Chat](/explore-chat) and [Pre-indexed Sources](/pre-indexed-sources).
  </Card>
</CardGroup>

| If your goal is... | Start here | Primary tools or pages |
|--------------------|------------|-------------------------|
| Bring a new source into Nia | Index & Subscribe | `index`, `manage_resource`, `auto_subscribe_dependencies` |
| Search or inspect what is already indexed | Search, Read & Explore | `search`, `nia_read`, `nia_grep`, `nia_explore` |

## Common Starting Flows

<Steps>
  <Step title="Bring in a Source">
    Index a repository, documentation site, paper, dataset, spreadsheet, or local folder with `index`, or use the dedicated [Google Drive Integration](/google-drive) flow for Drive content.
  </Step>

  <Step title="Search or Inspect It">
    Use `search` for semantic retrieval, `nia_grep` for pattern matching, `nia_read` to open content, and `nia_explore` to inspect structure.
  </Step>

  <Step title="Escalate to Research When Needed">
    Use `nia_research`, [Tracer](/tracer), or [Oracle Research Agent](/oracle-research) when you need synthesis, comparisons, or deeper investigation.
  </Step>

  <Step title="Save What Matters">
    Use `context` to save the findings, then reuse them in another agent or follow-up session.
  </Step>
</Steps>

## Tool Guide

<AccordionGroup>
  <Accordion title="Index & Subscribe">
    ### `index`
    Universal entry point for repositories, documentation, research papers, HuggingFace datasets, spreadsheets, and local folders.

    **Auto-detects**:
    - GitHub URLs as repositories
    - arXiv and PDF URLs as papers or PDFs
    - HuggingFace dataset URLs as datasets
    - CSV, TSV, XLSX, and XLS files as spreadsheets
    - Local paths as local folders
    - Other web URLs as documentation

    **Typical prompts**:
    ```
    "Index https://github.com/owner/repo"
    "Index https://docs.example.com"
    "Index https://arxiv.org/abs/2401.12345"
    "Index https://huggingface.co/datasets/openai/gsm8k"
    ```

    ### `auto_subscribe_dependencies`
    Parse a manifest such as `package.json`, `requirements.txt`, `pyproject.toml`, `Cargo.toml`, or `go.mod`, then subscribe or index related documentation sources automatically.

    **Best for**:
    - dependency onboarding
    - spinning up a project knowledge base quickly
    - keeping documentation context aligned with a codebase

    ### `manage_resource`
    List, check status, rename, delete, or subscribe to indexed resources.

    **Most common actions**:
    - `list`
    - `status`
    - `rename`
    - `delete`
    - `subscribe`
  </Accordion>

  <Accordion title="Search, Read & Explore">
    ### `search`
    Semantic search across indexed repositories, docs, papers, datasets, spreadsheets, Google Drive, and local folders.

    **Best for**:
    - answering natural-language questions
    - finding relevant sections before reading deeply
    - searching across multiple sources at once

    ### `nia_read`
    Read content from a repository, documentation source, package, Google Drive source, local folder, or HuggingFace dataset.

    ### `nia_grep`
    Regex search across repositories, documentation, packages, Google Drive sources, local folders, and datasets.

    ### `nia_explore`
    Browse trees and directories before reading individual files or rows.

    ### `get_github_file_tree`
    Inspect a GitHub repository structure without indexing it first.

    ### `nia_package_search_hybrid`
    Semantic search across public package source code without indexing first. Supports PyPI, npm, Crates.io, and Go modules.

    **Typical prompts**:
    ```
    "Search the requests package for how error handling works"
    "Find authentication examples in the fastapi package"
    ```
  </Accordion>

  <Accordion title="Research & Analysis Tools">
    ### `nia_research`
    AI-powered research with three modes for different levels of depth.

    | Mode | Best for |
    |------|----------|
    | `quick` | Fast web search and quick source discovery |
    | `deep` | Comparisons, evaluations, and multi-source analysis |
    | `oracle` | Complex multi-step investigations with autonomous research |

    ### `nia_advisor`
    Analyze your code against indexed documentation for grounded recommendations.

    **Typical prompts**:
    ```
    "Use nia_research deep to compare Prisma vs Drizzle"
    "Use nia_research oracle to investigate authentication best practices for Next.js"
    "Use nia_advisor to check my authentication code against the BetterAuth docs"
    ```

    **Related pages**:
    - [Tracer](/tracer)
    - [Oracle Research Agent](/oracle-research)
  </Accordion>

  <Accordion title="Vaults (Knowledge Base)">
    ### Vault Management
    Agent-maintained personal wikis that compound knowledge from your indexed sources. Create a vault, add sources, and the agent builds an interlinked wiki with concepts, entities, and a knowledge graph.

    **CLI commands**:
    ```bash
    nia vault init "Topic" --from-source <id1>,<id2>   # one-shot setup
    nia vault dream <vault-id>                          # self-improving dream cycle
    nia vault search <vault-id> "query"                 # semantic search
    nia vault open <vault-id>                           # interactive bash session
    ```

    **Workflows**: `ingest` (build pages from sources), `sync` (update stale pages), `lint` (find issues), `dream` (discover entities + connections), `refresh` (daily cron mode).

    **Related pages**:
    - [Vaults](/vaults)
    - [Dream Cycle](/vault-dream)
  </Accordion>

  <Accordion title="Context Sharing">
    ### `context`
    Save, retrieve, search, update, and delete cross-agent conversation context.

    **Memory types**:
    - `scratchpad` for short-lived working memory
    - `episodic` for session-level continuity
    - `fact` for persistent facts
    - `procedural` for reusable how-to knowledge

    **Typical prompts**:
    ```
    "Save this conversation about authentication setup"
    "Search my saved contexts for database migrations"
    ```
  </Accordion>
</AccordionGroup>
